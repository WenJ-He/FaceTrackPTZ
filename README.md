# FaceTrack PTZ

`FaceTrack PTZ` 是一个基于全景视频流进行人脸检测、驱动海康 PTZ 摄像头旋转变焦、在变焦过程中执行人脸识别并评估识别效果提升的人脸自动扫描项目。

当前仓库的首要目标不是直接写实现代码，而是先把需求、工程边界和后续代码约束固定下来。

## 文档入口

- 详细需求说明见 [docs/requirements.md](/mnt/e/hwj/FaceTrack%20PTZ/docs/requirements.md)

## 当前项目目标

- 从全景流中检测人脸
- 按全景从左到右、从上到下的固定方向依次触发 PTZ 旋转变焦
- 在旋转变焦过程中把阶段性人脸图送入人脸识别模型
- 基于数据库人脸库比对记录相似度变化和识别效果提升
- 在设备原生网页的视频画面上叠加人脸框与身份信息
- 输出识别过程报告，支持 `report`、`md` 或结构化 `log`

## 当前工程原则

- 检测、识别、PTZ 控制、网页叠加必须解耦
- 模型服务统一通过 Triton Inference Server 接入
- 检测链路优先保证低延迟和快速就绪，识别链路不能阻塞首轮检测启动
- 扫描方向固定为从左上到右下，当前轮不回头，走到最右下后下一轮再回到最左上
- 人脸库先存数据库，当前规模约 30 人，后续应可扩展
- 业务阈值、设备信息、模型名、数据库连接信息必须配置化
- 需要同时支持在线运行和离线评估两类场景
- 对“相似度提升”和“识别准确率”必须分开定义，不能混写

## 建议的近期开发顺序

1. 固化需求、目录和配置结构
2. 打通全景流读取
3. 接入 Triton 的检测模型
4. 确保检测模型预热、健康检查和低延迟启动
5. 复用既有 PTZ 控制逻辑
6. 接入 Triton 的识别模型与数据库人脸库检索
7. 记录每次变焦过程中的识别分数变化
8. 实现设备网页上的叠加层
9. 最后再做更复杂的跟踪和策略优化
